A ISO/IEC/IEEE 15289 é uma norma internacional que especifica itens de informação que devem estar incluídos na documentação ao longo do ciclo de vida de um software ou sistema. Essa norma assume que uma organização está seguindo uma das duas normas a seguir:

- [[ISO IEC IEEE 12207]], norma dos processos de ciclo de vida de um software.
- [[ISO IEC IEEE 15288]], norma dos processos de ciclo de vida de um sistema.

O objetivo da norma é fornecer uma diretriz para que seus usuários identifiquem e planejem os itens de informação específicos que serão desenvolvidos e revisados ao longo do ciclo de vida do software ou sistema.

Dois dos tópicos cobertos nessa norma são dados de ciclo de vida e itens de informação. São conceitos muito entrelaçados, mas antes de tudo é importante distingui-los.

Dados de ciclo de vida são dados **não-processados** vindos do software ou sistema. Dados coletados por sensores, históricos de transação, arquivos de log isolados, etc. Isolados, esses dados dificilmente geram informação de fato. Esses dados geralmente precisam ser agrupados e analisados para gerar informação de fato.

Itens de informação são dados **processados** e organizados de forma a carregarem alguma informação importante como parâmetro de entrada para a realização de atividades e tarefas de processos do software ou sistema. Carregam dados do ciclo de vida dentro de si.
